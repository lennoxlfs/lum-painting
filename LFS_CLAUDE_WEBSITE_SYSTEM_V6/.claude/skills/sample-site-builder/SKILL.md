# Sample Site Builder

Use this skill when the user wants a sample site concept, page plan, build prompt, implementation brief, or draft.

## Inputs
- Research Package
- Brand Package
- current website URL
- user design prompt / direction
- CTA goal
- relevant call notes

## Main objective
Create a unique-looking sample site that is easy to navigate, true to the brand, and stronger for local SEO and conversion.

## Non-negotiables
- custom-looking, not template-looking
- easy to navigate
- mobile-first
- stronger service clarity
- stronger local relevance
- stronger trust and polish
- stronger CTA path
- structurally SEO-safe
- aligned with the Google-first guidance in `reference/google-search-playbook-2026.md`
- compatible with the AI-era guidance in `reference/google-ai-search-playbook-2026.md`

## Output options
Depending on the user request, provide one or more of:
- sample site strategy
- sitemap / page plan
- homepage wireframe in words
- build prompt for another coding model
- actual section / copy outline
- implementation brief
- deployment-ready build notes
- comparison notes for report creation

## Rules
- follow the user's design direction closely
- preserve or elevate brand DNA unless told otherwise
- vary the visual feel project to project
- do not sacrifice clarity for flash
- solve the biggest problems from the Research Package


## Hand-off requirement
When the sample or final build is approved for implementation, organize output so it can move cleanly into the LFS launch path: QA -> GitHub push -> Vercel deploy.
