# Monthly Implementation Handoff

Use this skill when a client is on the monthly SEO evolution plan and the monthly reports need to become actual website updates.

## Required inputs
- `monthly-client-report.md` if available
- `monthly-internal-implementation-report.md`
- current analytics/reporting inputs
- current codebase
- any client approvals or exclusions for the month

## Purpose
Turn monthly SEO findings into approved, logged, deployable site changes.

## Workflow
1. Read `monthly-internal-implementation-report.md` first.
2. Extract approved edits, page targets, priority level, and validation steps.
3. Build a monthly implementation checklist.
4. Apply only the approved changes to the site.
5. Keep the site visually stable unless a visual change is explicitly approved.
6. Update `implementation-log.md` with this month’s changes.
7. Prepare a QA list, git-ready summary, and deployment note.

## Expected edit types
- title and meta updates
- internal linking updates
- page copy refinements for service clarity and local relevance
- schema updates
- CTA/trust tweaks
- performance cleanup
- image alt text and accessibility fixes
- FAQ or content additions where approved

## Output requirements
Always produce:
1. monthly implementation checklist
2. code/content changes
3. implementation-log update
4. short QA + validation plan for next month

## Guardrails
- do not implement unapproved major redesigns
- do not turn the monthly plan into arbitrary changes
- keep changes tied to the report findings and the business goal
