# Research Package Builder

Use this skill when the user wants the Step 2A Research Package.

## Required inputs
- current website URL
- Google Business Profile link
- 3 competitor URLs
- services
- service area
- social links
- call notes
- Lighthouse report summary
- PageSpeed report summary
- Rich Results summary
- GBP audit summary

If some inputs are missing, proceed with what is available and clearly label assumptions or gaps.

## Objectives
Produce a package that helps:
1. build the sample site correctly
2. build the client-facing sales report correctly

## Use this logic
- prioritize issues that visibly affect trust, clarity, search relevance, conversion, or UX
- separate measured findings from inferred findings
- include the pages or areas the sample must fix first
- tie SEO findings to business outcomes where possible
- use the diagnostics stack in `reference/diagnostics-stack.md`

## Output structure
1. Business Summary
2. SEO / Local SEO Findings
3. Speed / Technical Findings
4. Competitor Notes
5. Conversion Notes
6. Weakness List
7. Opportunity List
8. Strategy for Sample Site
9. Strategy for Sales Report
