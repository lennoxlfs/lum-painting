# Monthly Action Engine

## Purpose
Turn monthly Search Console, GA4, GBP, Lighthouse, and PageSpeed findings into a prioritized action plan and, when asked, concrete site edits.

## Use when
- monthly reports are available
- the user wants SEO changes based on analytics
- the user wants to know what to edit next
- the user wants a month-over-month action plan

## Inputs
- monthly growth report
- Search Console export or summary
- GA4 landing-page or event summary
- GBP performance notes if available
- Lighthouse or PageSpeed findings if relevant
- current site URL or page list

## Required behavior
1. Identify the few changes that matter most.
2. Separate visibility, CTR, engagement, conversion, and technical issues.
3. Translate each important issue into an edit category.
4. Prioritize actions by business impact and difficulty.
5. Recommend exact page-level changes whenever enough evidence exists.
6. State what should be measured next month to validate the edits.

## Output structure
1. Executive Summary
2. Top Findings
3. Priority Actions
4. Proposed Edits This Month
5. Validation Plan For Next Month

## Edit categories
- title/meta refinement
- heading and topical clarity refinement
- service/city relevance refinement
- internal linking improvements
- schema improvements
- CTA and trust placement improvements
- GBP freshness improvements
- performance cleanup

## Guardrails
- do not overreact to one noisy data point
- distinguish indexing issues from ranking issues
- distinguish CTR issues from conversion issues
- distinguish low traffic from low lead quality
- state uncertainty when confidence is low
