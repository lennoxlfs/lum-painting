# Metrics and Reporting Map

## Search Console

Use Search Console for:
- clicks
- impressions
- CTR
- average position
- page-level and query-level visibility
- search type splits
- indexing and URL inspection
- Insights summaries
- branded vs non-branded analysis when available

Best use:
- diagnose visibility and CTR opportunities
- identify the pages and queries worth improving first

## GA4

Use GA4 for:
- landing-page traffic
- engagement rate
- engaged sessions
- key events / conversions
- source/medium context
- pages with traffic but weak action
- pages with strong engagement but weak conversion

Best use:
- diagnose user quality and conversion quality
- see what organic traffic does after it lands

## Google Business Profile

Use GBP Performance for:
- views
- calls
- website clicks
- direction requests
- profile interaction trends

Best use:
- local discovery and action tracking
- website/profile alignment checks

## Lighthouse / PageSpeed / Core Web Vitals

Use for:
- page speed
- render / asset issues
- UX friction
- layout shift
- responsiveness
- major page-experience problems

Best use:
- fix issues that suppress usability and conversions
- validate performance-sensitive pages

## LFS monthly metric priorities

### Primary
- organic conversions / key events
- calls / form fills / bookings
- Search Console clicks and impressions
- landing-page engagement

### Secondary
- average position
- GBP actions
- page speed / CWV issues on important pages

### Tertiary
- vanity traffic spikes with no business value

## Interpretation rules

- Do not celebrate traffic if conversions are flat and traffic quality is poor.
- Do not panic over tiny position changes on already strong pages.
- Do not make radical changes after a small fluctuation.
- Compare search data in context, not in isolation.
- Separate branded demand from non-branded discovery where possible.


## Vercel analytics layer
- Vercel Web Analytics: traffic and page behavior trend checks
- Vercel Speed Insights: Core Web Vitals and performance trend checks
- Use both with GA4 and Search Console, not instead of them
