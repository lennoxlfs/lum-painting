# Step 2A Diagnostics Stack

## Always collect before the meeting when possible

### 1. Lighthouse
Run on:
- homepage
- main service page
- contact page
- one important location/service page if applicable

Capture:
- Performance
- Accessibility
- Best Practices
- SEO
- top 3 to 5 issues
- top opportunities

### 2. PageSpeed Insights
Run on:
- homepage
- main service page
- contact page

Capture:
- mobile score
- desktop score
- top opportunities
- major rendering or asset issues
- any serious mobile warnings

### 3. Rich Results Test
Run on:
- homepage
- main service page
- contact page

Capture:
- schema types found
- errors
- warnings
- whether markup matches the visible content
- missing useful business markup

### 4. Public GBP audit
Capture:
- category quality
- service clarity
- hours
- reviews and freshness
- photo quality and freshness
- website link quality
- profile completeness

### 5. Competitor note pass
Review:
- 3 competitors
- strongest pages
- stronger trust or CTA patterns
- stronger local positioning
- stronger service clarity

## Access-only extras after close or when granted

### Search Console
Use:
- Performance
- Page indexing
- URL Inspection
- Core Web Vitals
- Insights

### GA4
Use:
- Landing page report
- Engagement metrics
- Key events / conversions
- source/medium by landing page

### GBP Performance
Use:
- views
- calls
- website clicks
- direction requests

## Output
All of the above should feed the Step 2A Research Package.
