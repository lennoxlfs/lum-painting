# Monthly Growth Loop

This is the LFS monthly SEO and conversion adjustment system.

## Goal

Use monthly reports to decide what changes should be made to keep SEO and business performance moving in the right direction.

## Inputs

Preferred monthly input stack:
- Search Console Performance export
- Search Console Insights summary
- GA4 Landing Page report
- GA4 engagement / key-event metrics
- GBP Performance
- Lighthouse or PageSpeed reruns for pages that matter
- any sales / call / form notes from the business
- any recent edits or promotions

## Monthly loop

### 1. Measure
Collect:
- clicks
- impressions
- CTR
- average position
- landing-page sessions
- engagement rate
- key events / conversions
- calls / forms / bookings
- GBP website clicks / calls / direction requests
- Core Web Vitals or page-speed issues when relevant

### 2. Diagnose
Separate the issue type:

#### A. Technical / indexing problem
Signals:
- page disappears from search
- indexing drops
- huge page-level impression loss
- crawl or canonical issues
- broken pages
Action:
- investigate indexing, canonicals, noindex, redirects, render issues

#### B. Visibility problem
Signals:
- impressions stagnate or fall
- average position weak
- non-branded discovery weak
Action:
- strengthen relevance, internal linking, service clarity, local relevance, supporting content

#### C. CTR problem
Signals:
- impressions healthy but clicks lag
- pages rank but are not winning the click
Action:
- improve titles, snippets, messaging clarity, visible trust, page/topic match

#### D. Engagement problem
Signals:
- traffic arrives but engagement rate is weak
- users do not view additional pages or trigger key events
Action:
- improve first screen, page clarity, speed, content usefulness, CTA sequence

#### E. Conversion problem
Signals:
- page gets traffic and engagement, but low lead activity
Action:
- strengthen CTA placement, offer clarity, forms, phone visibility, trust, urgency, follow-up actions

#### F. Local-profile problem
Signals:
- weak GBP actions or stale profile
Action:
- improve profile completeness, category alignment, photos, review responses, service alignment

### 3. Change
Pick the fewest high-impact changes first.
Default monthly priority order:
1. technical blockers
2. high-impression / low-click pages
3. high-traffic / low-conversion pages
4. top services with weak local visibility
5. stale trust/profile assets
6. expansion opportunities

### 4. Validate
After changes:
- annotate what changed
- compare against the prior period
- check if impressions, CTR, engagement, or key events improved
- if not, do not keep guessing blindly; reassess the problem type

## Recommended monthly output

1. What the data is saying
2. Top 3 priorities
3. Exact changes to make
4. Why those changes are first
5. What to watch next month

## LFS decision rules

### If impressions are up but CTR is weak
Likely problem:
- weak snippet appeal
- weak perceived relevance
Do first:
- improve titles and meta descriptions
- tighten H1 / first-screen message
- improve trust cues seen immediately after landing

### If CTR is fine but positions are weak
Likely problem:
- content relevance / depth
- internal linking
- local specificity
Do first:
- strengthen page usefulness
- improve supporting sections
- add or improve internal links
- clarify service-area fit

### If traffic is up but conversions are weak
Likely problem:
- conversion path
- CTA sequence
- trust
Do first:
- simplify CTA flow
- raise phone/form visibility
- add social proof / reassurance
- reduce friction

### If branded traffic is fine but non-branded is weak
Likely problem:
- poor discovery for new searchers
Do first:
- strengthen non-branded service pages
- expand useful service support content
- improve local service relevance

### If GBP actions are weak
Likely problem:
- profile or website support mismatch
Do first:
- refresh profile details and photos
- align website services and location signals
- improve review response and trust freshness
