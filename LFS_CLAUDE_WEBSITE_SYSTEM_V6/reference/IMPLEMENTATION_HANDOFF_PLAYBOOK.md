# Implementation Handoff Playbook

This playbook defines how strategy documents turn into live website changes.

## Principle
Client-facing reports are for communication.
Internal reports are for Claude Code implementation.

## Final build handoff
Use these files as the build brief:
- research-package.md
- brand-package.md
- internal-sales-report.md

### Final build sequence
1. Read the three handoff files
2. Convert them into an implementation checklist
3. Apply changes in the codebase
4. Update implementation-log.md
5. Run QA
6. Push to GitHub and deploy to Vercel

## Monthly handoff
Use these files as the monthly update brief:
- monthly-client-report.md
- monthly-internal-implementation-report.md

### Monthly sequence
1. Read the internal monthly report
2. Confirm which changes are approved
3. Convert them into an implementation checklist
4. Apply the approved changes
5. Update implementation-log.md
6. Run QA and validation
7. Push to GitHub and deploy to Vercel

## Standard filenames
- research-package.md
- brand-package.md
- client-sales-report.md
- internal-sales-report.md
- monthly-client-report.md
- monthly-internal-implementation-report.md
- implementation-log.md
